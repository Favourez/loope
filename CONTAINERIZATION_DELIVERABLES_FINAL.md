## üìà **DELIVERABLE 3: SCALING, ROLLING UPDATES, AND SERVICE DISCOVERY** ‚úÖ

### **‚úÖ Horizontal Pod Autoscaler (HPA)**

#### **Auto-Scaling Configuration**
```yaml
# Emergency App HPA
spec:
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
  - type: Resource
    resource:
      name: memory
      target:
        type: Utilization
        averageUtilization: 80
```

**Scaling Triggers:**
- **CPU > 70%**: Scale up automatically
- **Memory > 80%**: Scale up automatically
- **Low traffic**: Scale down after stabilization period

### **‚úÖ Rolling Update Strategy**

#### **Zero-Downtime Deployments**
```yaml
strategy:
  type: RollingUpdate
  rollingUpdate:
    maxSurge: 1        # Add 1 extra pod during update
    maxUnavailable: 1  # Allow 1 pod to be unavailable
```

**Update Process:**
1. Create new pod with updated image
2. Wait for readiness probe to pass
3. Terminate old pod gracefully
4. Repeat until all pods updated
5. **Zero downtime guaranteed**
